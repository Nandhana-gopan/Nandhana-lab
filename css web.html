WHAT IS CSS

CSS, or Cascading Style Sheets, is a fundamental technology used in web development to control the visual appearance and layout of web pages. While HTML (HyperText Markup Language) provides the basic structure and content of a webpage—such as headings, paragraphs, images, and links—CSS is what brings that content to life by defining how it looks to the user. With CSS, developers can apply styles to HTML elements, including colors, fonts, spacing, positioning, and even complex animations, creating a polished and engaging user experience. One of the key strengths of CSS is its ability to separate content from design, allowing developers and designers to make changes to the visual style of an entire website by modifying just a single stylesheet file, rather than editing each HTML page individually. This separation also improves website maintenance, scalability, and consistency. The term “cascading” in CSS refers to the way styles are applied and prioritized—styles can come from multiple sources such as browsers’ default settings, external stylesheets, inline styles, and user preferences—and the browser resolves conflicts by following a specific hierarchy to determine which styles take precedence. CSS supports a broad range of features from basic styling like text color and font size to advanced layout techniques using Flexbox and CSS Grid, enabling responsive designs that adapt seamlessly to different screen sizes and devices. Furthermore, CSS has evolved over the years to include powerful tools for animations, transitions, and interactive effects, all without the need for JavaScript. Overall, CSS is an essential part of modern web design, empowering developers to create visually stunning, accessible, and user-friendly websites that work consistently across all browsers and devices.

WHY USE CSS ?

Key aspects of CSS include:

  • Styling and Layout: CSS controls the visual appearance and layout of web page elements. This includes properties like fonts, colors, spacing, dimensions, positioning, and responsiveness for different screen sizes.
  • Separation of Concerns: A core principle of CSS is the separation of content (HTML) from presentation (CSS). This makes websites easier to maintain, update, and more flexible for different display media (e.g., screen, print).
  • Cascading and Specificity: The "Cascading" in CSS refers to the rules that govern how styles are applied when multiple rules might conflict. This involves a system of specificity and inheritance to determine which styles take precedence.
  • Syntax: CSS rules consist of a selector (targeting HTML elements) and a declaration block. The declaration block contains one or more declarations, each with a property (e.g., color, font-size) and its corresponding value (e.g., blue, 16px).
  • Consistency: CSS ensures uniform design across multiple web pages by using stylesheets applied site-wide.
  • Flexibility: CSS makes it easy to change the look and feel of a website without touching the HTML structure.
  • Improved load times: External CSS files can be cached by browsers, improving page load speeds.
  • Responsive design: CSS enables websites to adapt to different screen sizes and devices (mobile, tablets, desktops).

HOW TO ADD CSS INTO A FILE

There are three ways in which CSS can be used in a page.

1. Inline CSS - Inline CSS involves applying style rules directly to individual HTML elements via the style attribute.

When to use it:

  1. Great for quick, one-off styling or testing.
  2. Not recommended for maintaining or scaling a project.


Pros: Immediate application, easy to test.

Cons: Makes HTML cluttered and repetitive; poor maintainability.


2. Internal CSS (Embedded CSS)- Internal CSS is placed within a style tag in the head section of an HTML document.

When to use it: Useful for single-page applications or quick prototypes.


Pros: Keeps all styles in one place within a single document; better than inline for organization.

Cons: Styles are confined to that one page and can become bloated with complexity.


3.External CSS- External CSS lives in a separate .css file, connected to the HTML via a link tag

When to use it: Perfect for websites with multiple pages or large-scale projects.


Pros: Promotes separation of content and style, easier maintenance, reusable across pages.

Cons: Slight initial load overhead; risk of a flash of unstyled content (FOUC) if CSS doesn't load swiftly.

EXAMPLE OF CSS SYNTAX:

h1
{
color: navy;
text-align: center;
}

p
{
font-family: sans-serif;
line-height: 1.5;
}

CSS SYNTAX BASICS

  • Selectors: Can be element names, classes, IDs, or more complex selectors.


  • Properties: Define what aspect of the element to style (color, margin, padding, etc.).


  • Values: Specify the exact setting for the property.
